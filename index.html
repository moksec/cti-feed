The SSHStalker botnet represents a significant and widespread threat to Linux-based infrastructures, particularly those in cloud environments. Its 'scale-first' operational model, combining noisy but effective methods like SSH brute-forcing with resilient, low-cost IRC-based command-and-control, indicates a persistent and adaptable adversary. The botnet's diverse monetization capabilities, including cryptomining, AWS key harvesting, and potential DDoS, suggest a flexible threat likely driven by financial gain. Organizations globally, especially those relying on Linux servers in cloud settings, should consider this an active and evolving risk.
Organizations should immediately implement and enforce strong SSH security practices, including disabling password authentication in favor of key-based authentication, and regularly rotating SSH keys. Comprehensive egress filtering should be deployed to prevent outbound IRC connections and block C2 communications. Security monitoring should be enhanced to detect unusual processes, specifically the installation or execution of compilers (e.g., GCC) on production servers, and the creation of cron jobs with short execution cycles, especially from temporary or unusual paths like `/dev/shm`. Proactive patching of Linux systems to address known vulnerabilities, regardless of age, is essential. For cloud environments, particular attention should be paid to monitoring for unauthorized AWS key harvesting attempts and anomalous activity within cloud accounts. Consider hardening production images by removing unnecessary developer tools like compilers.
SSHStalker is a newly documented Linux botnet that utilizes IRC for its command-and-control (C2) communications. Initial access is achieved through automated SSH scanning and brute-forcing, employing a Go binary disguised as 'nmap'. The botnet exhibits worm-like propagation, using compromised hosts to scan for additional SSH targets, with activity noted against Oracle Cloud infrastructure. Upon infection, it downloads the GCC tool to compile C-based IRC bots and other bot variants (GS, bootbou) on the victim host. Persistence is maintained via cron jobs set to run every 60 seconds. Privilege escalation is attempted using exploits for 16 undisclosed CVEs targeting Linux kernel versions from 2009-2010. Monetization capabilities include AWS key harvesting, website scanning, and cryptomining (e.g., PhoenixMiner for Ethereum). DDoS capabilities are present but have not been observed in active use.
