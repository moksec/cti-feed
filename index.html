EDR killer tool uses signed kernel driver from forensic software - bleepingcomputer.com
Financially motivated threat actors, likely linked to ransomware operations, are actively deploying sophisticated EDR bypass techniques that leverage legitimate but vulnerable signed kernel drivers. This approach undermines the efficacy of endpoint security solutions, posing a significant risk to organizational defenses by enabling attackers to disable critical security controls and establish stealthy persistence. The abuse of legitimate tools and the continued vulnerability of Windows' driver signature enforcement mechanism for older certificates represent a persistent challenge for overall security posture, demanding a multi-layered defense strategy.
Organizations should immediately implement and enforce Multi-Factor Authentication (MFA) across all remote access services, especially VPNs, to mitigate initial access vectors. Proactive monitoring of VPN logs for suspicious activity is crucial for early detection. To counter the EDR bypass technique, enable Hypervisor-Protected Code Integrity (HVCI) and Memory Integrity, which help enforce Microsoft’s vulnerable driver blocklist. Furthermore, security teams should actively monitor for any kernel services masquerading as OEM or hardware components and deploy Windows Defender Application Control (WDAC) and Attack Surface Reduction (ASR) rules to block known vulnerable signed drivers. Regularly auditing and updating security configurations and EDR solutions is also essential.
A custom EDR killer tool, disguised as a firmware update utility, was observed in an incident, exploiting compromised SonicWall SSL VPN credentials lacking MFA. This tool utilizes the 'Bring Your Own Vulnerable Driver' (BYOVD) technique, specifically abusing 'EnPortv.sys,' an old 64-bit kernel driver from EnCase forensic software. Despite its certificate expiring in 2010 and being revoked, Windows systems still accept it due to timestamp-based validation and an exception for certificates issued before July 29, 2015. The driver is installed as a fake OEM hardware service for reboot-resistant persistence. It leverages the driver’s kernel-mode IOCTL interface to terminate 59 targeted EDR and antivirus processes, bypassing Protected Process Light (PPL) by executing a kill loop every second. Initial reconnaissance included ICMP ping sweeps, NetBIOS name probes, and SMB activity, with SYN flooding exceeding 370 SYNs/sec.
