New ClickFix attacks abuse Windows App-V scripts to push malware - bleepingcomputer.com
This report details a sophisticated, financially motivated campaign leveraging the 'ClickFix' method, fake CAPTCHA, and abuse of Windows App-V scripts to deliver the Amatera infostealer. The attack chain exhibits advanced evasion techniques, including sandbox detection, use of legitimate Windows components (living-off-the-land binaries), and steganography. The Amatera infostealer, a malware-as-a-service (MaaS) offering, poses a significant and evolving threat to organizations as it aims to exfiltrate sensitive browser data and credentials. Organizations should recognize the escalating complexity of such attacks and prioritize user awareness, robust endpoint security, and proactive threat detection strategies to mitigate the risk of data theft.
To defend against these types of attacks, IT and security management should consider the following actions: 1) Enhance user training on social engineering tactics, particularly fake CAPTCHA prompts, and the dangers of executing untrusted commands. 2) Implement Group Policy to restrict or disable access to the Windows Run dialog if not essential for business operations. 3) Remove or disable Microsoft App-V components (e.g., `SyncAppvPublishingServer.vbs`) on endpoints where they are not explicitly required, thereby reducing the attack surface. 4) Ensure comprehensive PowerShell logging is enabled across all endpoints to aid in the detection and investigation of malicious PowerShell activity. 5) Deploy and configure network monitoring solutions to identify suspicious outbound connections, specifically looking for mismatches between HTTP Host headers or TLS Server Name Indication (SNI) and destination IP addresses, which can indicate C2 communication. 6) Strengthen Endpoint Detection and Response (EDR) capabilities to detect living-off-the-land abuse, in-memory execution, and steganographic payload delivery.
The attack initiates with a fake CAPTCHA instructing victims to manually execute a command via the Windows Run dialog. This command abuses the legitimate `SyncAppvPublishingServer.vbs` App-V script, executed via `wscript.exe`, to launch PowerShell. Initial stages include anti-analysis checks, stalling execution with infinite waits if a sandbox is detected. Configuration data is retrieved from public Google Calendar events (base64-encoded). The infection chain spawns a hidden 32-bit PowerShell process via Windows Management Instrumentation (WMI), decrypts and loads embedded payloads into memory. It then employs LSB steganography to embed encrypted PowerShell payloads within PNG images hosted on public CDNs, retrieved dynamically using WinINet APIs. The payload is extracted, decrypted, GZip-decompressed, and executed in memory. The final PowerShell stage decrypts and launches native shellcode, which executes the Amatera infostealer. Amatera, based on ACR infostealer and available as MaaS, connects to a hardcoded IP address for endpoint mappings and receives additional binary payloads via HTTP POST to exfiltrate browser data and credentials.
