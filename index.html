Multiple critical vulnerabilities, collectively tracked as CVE-2026-25049, exist in n8n, an open-source workflow automation platform. These flaws allow any authenticated user with workflow creation/editing privileges to achieve unrestricted Remote Code Execution (RCE) and full control over the host server. The vulnerabilities stem from incomplete sanitization mechanisms and weak sandboxing of user-written server-side JavaScript expressions, effectively bypassing previous patches for CVE-2025-68613. Exploitation can lead to the compromise of n8n instances, arbitrary system command execution, theft of stored credentials (API keys, OAuth tokens), filesystem access, and potential pivoting to connected cloud accounts and AI workflows. The issue involves a type-confusion vulnerability where runtime checks do not enforce TypeScript typings. Public Proof-of-Concept (PoC) exploits are available from multiple security researchers (Pillar Security, Endor Labs, SecureLayer7). Affected versions include all prior to 2.5.2 and 1.123.17.
